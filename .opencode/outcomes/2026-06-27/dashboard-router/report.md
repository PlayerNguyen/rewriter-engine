# Dashboard Router — TanStack Router Setup

**Date:** 2026-06-27
**Branch:** `feat/dashboard-router`

## What was done

Set up file-based routing for the dashboard app using TanStack Router v1.170.

### Packages installed

- `@tanstack/react-router` — core router
- `@tanstack/router-plugin` — Vite plugin for file-based route code generation
- `@tanstack/router-devtools` — optional devtools

### Route structure

```
apps/dashboard/src/routes/
  __root.tsx       # Root layout: DashboardLayout + Sidebar + <Outlet />
  index.tsx        # "/" — Home analytics dashboard
  articles.tsx     # "/articles"
  rewrites.tsx     # "/rewrites"
  sources.tsx      # "/sources"
  prompts.tsx      # "/prompts"
  settings.tsx     # "/settings"
  logs.tsx         # "/logs"
```

### Sidebar routing

- Added `to` field to `SidebarLeaf` type in `@rewriter/ui`
- Added `linkComponent` prop to `Sidebar` — accepts TanStack Router's `<Link>` for client-side navigation
- Added standalone "Home" leaf at top of sidebar (no group)
- Sidebar items render as proper `<a>` elements via `<Link>`
- Active highlighting computed from `useRouterState().location.pathname`
- Groups auto-expand only when a child route is active; collapse otherwise
- Switched sidebar item search from `<input>` to `<CommandInput>` component (search is inside sidebar)
- Search renders a `Icon` component

### Key decisions

| Decision | Rationale |
|---|---|
| File-based routing | Recommended by TanStack; cleaner DX |
| Flat route paths (`/articles`, etc.) | Sidebar groups are visual-only, not route hierarchy |
| `linkComponent` prop on Sidebar | Keeps shared UI package router-agnostic |
| Active state via `useRouterState` | Computed in `__root.tsx`, passed as `active` prop |
| `useEffect` syncs `expandedGroups` to `defaultExpanded` | Groups collapse on navigation to irrelevant sections |
