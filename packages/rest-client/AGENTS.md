# @rewriter/rest-client — Orval-Generated API Client

Auto-generated type-safe HTTP client from the server OpenAPI spec (`apps/server/openapi.json`). Uses native `fetch` via a custom mutator that handles base URL resolution, auth injection, and error normalization.

## Quick Start

```typescript
import { getApiV1Table, getApiV1Health } from "@rewriter/rest-client";
import type {
  GetApiV1Table200,
  GetApiV1TableParams,
  GetApiV1Health200,
} from "@rewriter/rest-client";

// Health check
const health: GetApiV1Health200 = await getApiV1Health();

// Query a registered table
const result: GetApiV1Table200 = await getApiV1Table({
  id: "sources",
  page: 1,
  limit: 10,
  sort: JSON.stringify({ fieldName: "createdAt", direction: "desc" }),
  search: "keyword",
});
```

```

├── orval.config.ts                  # Orval config (input spec, output paths, mutator)
├── scripts/
│   └── generate-barrel.ts           # Post-generation barrel exporter
└── src/
    ├── index.ts                     # Public barrel exports
    ├── mutator/
    │   ├── fetch-instance.ts        # customFetchInstance mutator
    │   └── fetch-instance.test.ts   # Mutator unit tests
    └── generated/                   # Auto-generated by Orval (do not edit manually)
        ├── api/
        │   ├── index.ts             # Barrel of all API tag modules
        │   ├── health/health.ts     # getApiV1Health
        │   └── table/table.ts       # getApiV1Table
        └── model/
            ├── index.ts             # Barrel of all model types
            ├── getApiV1Health200.ts
            ├── getApiV1Table200.ts
            ├── getApiV1Table400.ts
            ├── getApiV1Table404.ts
            └── getApiV1TableParams.ts
```

## Generation

```bash
# Regenerate from the current OpenAPI spec
bun run --filter @rewriter/rest-client generate

# This runs:
#   1. orval --config ./orval.config.ts   — generates API functions + model types
#   2. bun run scripts/generate-barrel.ts  — regenerates src/generated/api/index.ts
```

The input spec is `../../apps/server/openapi.json`. The generated files in `src/generated/` are **never** edited by hand — they are regenerated each time and `clean: true` removes stale outputs.

## Exports

| Export | Kind | Description |
|--------|------|-------------|
| `getApiV1Health` | function | `GET /api/v1/health` — health check |
| `getApiV1Table` | function | `GET /api/v1/table` — query a registered table |
| `GetApiV1Health200` | type | Health check response shape |
| `GetApiV1Table200` | type | Table response (`data`, `total`, `page`, `limit`, `totalPages`) |
| `GetApiV1TableParams` | type | Table query params (`id`, `page?`, `limit?`, `sort?: string`, `search?`, `filters?: string`) |
| `GetApiV1Table400` | type | Validation error response |
| `GetApiV1Table404` | type | Not-found error response |
| `customFetchInstance` | function | Underlying fetch mutator (exported for advanced use) |
| `RequestParams` | type | Mutator input shape |

## Mutator Behavior

The `customFetchInstance` mutator is injected by Orval at generation time (`override.mutator`). Every generated API function delegates to it.

### Base URL Resolution

Priority order:
1. `VITE_API_URL` env var (for Vite-based apps like the dashboard)
2. `API_URL` env var (for Node/Bun runtimes)
3. `http://localhost:3001` (dev fallback)

### Auth Injection

In browser environments (`typeof document !== "undefined"`), reads `token` from `localStorage` and sets `Authorization: Bearer <token>`. Skipped entirely in server-side execution.

### Error Normalization

Non-2xx responses are thrown as `{ status: number, body: unknown }`:
- JSON response bodies are parsed and attached as `body`
- Non-JSON error bodies set `body: null`

### Request Features

- **Content-Type**: defaults to `application/json`; overridable via custom headers
- **Body serialization**: JSON-stringified before sending
- **Query params**: appended as URL search params; null/undefined values are skipped
- **Cancellation**: accepts `signal?: AbortSignal` for `AbortController` integration
- **Custom headers**: merged on top of defaults, custom `Content-Type` overrides the default

## Design Decisions

| Decision | Rationale |
|----------|-----------|
| Orval code generation | Type-safe API client from the single source of truth (OpenAPI spec); eliminates hand-written HTTP clients and type drift |
| Native `fetch` (not axios) | Zero runtime dependencies; fetch is universal across Bun/Node/browsers |
| `customFetchInstance` mutator | Single place for base URL, auth, error handling, serialization — Orval injects it into every generated function |
| `mode: 'tags-split'` | One file per OpenAPI tag; scales as endpoints are added |
| `clean: true` | Removes stale generated files when endpoints are deleted |
| `export type *` for models | Models are type-only exports — no runtime import cost |
| Barrel generation script | Orval does not auto-generate the API barrel with tags-split mode; post-generation script creates it |
| `signal` in `RequestParams` | `AbortSignal` passthrough to native `fetch` enables `AbortController` cancellation in hooks and consumers |
